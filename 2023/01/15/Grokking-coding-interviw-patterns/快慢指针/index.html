快慢指针

发表于2023-01-15|更新于2023-01-16|Coding Patterns快慢指针
|字数总计:1.9k|阅读时长:8分钟

快慢指针

  • 通常用来遍历iterable data structure。主要用来区分有向数据结构特征上（Linked list or array）。
  • 两个指针用于遍历array 或者list在任意方向上。通常一个速度是1一个是2.但是具体速度可按照问题调整
  • 不像双指针问题（关注数据值），快慢指针方法主要用数组中的索引或者链表中的结点指针来确定数据结构特征（比如说：判断数据结构中是否有圆）

是否问题可以使用快慢指针

满足其中之一就可以使用

  • 解决问题的过程中需要知道
    1. 第一个 x% of elements 在链表里
    2. 链表中第k个点的元素(比如：中间元素，第二个四分区间上开始的元素)
    3. 链表中倒数第k个元素
  • 解决问题需要检测链表中是否有环
  • 解决问题需要知道符号序列（sequence symbol）中是否有环

满足其一就不可以使用

  • 输入的数据不能线性遍历（不是array,linked list或者string）
  • 问题可以用双指针在相同的pace情况下解决

现实应用场景

  • 符号连接（symbolic link）验证：symbolic link是一个file point to another file.但是symlink很容易产生loops或者cycles当文件互相指向的时候。快慢指针用来
  • 编译面向对象的程序：对于大型应用，需要拆分大型模块为了更好的维护，依赖关系用来指定编译文件顺序。这个顺序可能是环形引用顺序。所以快慢指针用来判断环形引用终止程序。

Happy Number

Leetcode链接

朴素想法

不断计算digits的平方和然后放进hash set里面。对于每次计算，都要check计算的数字是否在set里面。如果存在说明找到了cycle就return false.没找到就加到hash set里面。知道converge到了1就return True
Time complexity O(logn)因为数据位数由logn来决定。在最坏的情况下，这些成本将是 O(log n) + O(log(log n))+…所以是O(logn)
Space complexity O(n)

快慢指针

  • 因为不是快乐数最终会陷入循环所以可以使用双指针
  • 首先定义function从输入的数字里构建单个digit的平方和。
    1. 首先要对数字取模%10
    2. 数字向下取整除10获取十位百位等
    3. 对取模等数字进行平方累加
  • 对于每一个iteration，如果fast不等于1的话，就让快指针走两步慢指针走一步slow =sum_digits(slow), fast = sum_digits(sum_digits(fast))
    Time Complexity O(logn + logn) = O(logn) two pointers
    Space Complexity O(1)

Linked List Cycle(检查环形链表是否有环)

Check if a linked list contains a cycle or not. If a cycle exists, return TRUE. Otherwise, return FALSE.

Naive approach

遍历链表的时候保存node到hash set里面。如果存在就说明发现了一个loop.否则遍历完全部直接直接return false

Time complexity O(n)
Space complexity worst case O(n)

快慢指针

  • 快慢指针都initial成head
  • 当终止条件fast.next和fast不等于None的时候一直执行
  • slow走一步，fast两步，当slow等于fast的时候就return True否则遍历到终点就return False

Time complexity O(n)
Space complexity O(1)

Circular Array Loop

Leetcode链接

% 运算符，取余数时余数符号由被除数决定

快慢指针

for loop 遍历数组的元素。对于每个元素判断是否有循环。

  • 移动慢指针如果慢指针违反条件（自循环或者方向和前面记录的方向相反）
  • 移动快指针一次，判断是否违反条件，再次移动快指针。再次判断是否违反条件
  • 终止条件：两个指针相等，同时slow或者fast其中一个不是-1
    令一难点：对于移动索引越界的话直接取模获得移动的位置。

Time complexity O(n2) or
O(n) Leetcode未懂
Space complexity O(1)

Middle of the Linked List(链表中间节点)

Given a singly linked list, return the middle node of the linked list. If the number of nodes in the linked list is even, return the second middle node.

快慢指针

fast指针：偶数会移动到Null,奇数会移动到最后一个node.

当快指针走到null或者最后一个node的时候，slow指针就在mid.(对于even来说在second mid)

Time complexity O(n)
Space complexity O(1)

Palindrome Linked List

Leetcode链接
For the given head of the linked list, find out if the linked list is a palindrome or not. Return TRUE if the linked list is a palindrome. Otherwise, return FALSE.

朴素想法

用stack遍历数组存储相同的就抵消。stack特性但是空间复杂度是O(n)

快慢指针+双指针

  • 向前面一样快慢指针找到链表中点
    这里有点需要记住，链表node是奇数时，fast指针停在最后一个node上。偶数时，fast停在none上。当为奇数是也就是fast!= None的时候需要移动slow指针一位，不判断mid节点
  • 反转slow指针之后的链表
  • 和前面半段链表进行判断 while second != None

Time complexity O(n)
Space complexity O(1)
